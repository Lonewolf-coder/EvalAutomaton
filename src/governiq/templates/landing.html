How Evaluation Works

Six-step dual-pipeline evaluation powered by structural analysis and live conversation testing.

Candidate Submits Bot Export
The candidate exports their chatbot from the Kore.ai XO Platform as a JSON file and uploads it along with their JWT credentials and Mock API URL.
Pipeline A — Structural Analysis (CBM Audit)
The engine parses the bot export and checks it against the assessment manifest: dialogs, entity nodes, Agent Nodes, service nodes (POST, GET, PUT, DELETE), and FAQ completeness. Deterministic and reproducible.
Pipeline B — Live Conversation Testing (Webhook)
An LLM-powered conversation driver talks to the bot through its webhook using JWT authentication. It tests CREATE, RETRIEVE, MODIFY, DELETE, and EDGE_CASE patterns — exactly like a human QA tester.
API State Verification
After each conversation, the State Inspector checks the candidate's Mock API to verify data was actually persisted correctly. A bot can say the right things but fail to save data — this catches that.
Compliance & Platform Insights
Platform configuration is verified via CBM and Kore.ai Public APIs. Bot analytics, intent detection stats, and NLP performance are captured. Critical failures override the overall score.
Detailed Report Generated
Both pipelines merge into a single scorecard. Every finding is explained in plain English with evidence cards. The candidate sees what passed, what failed, and what to fix.

Six Engine Patterns

Every task maps to one pattern. The engine is domain-agnostic.

CREATECollect entities, POST record
AMENDCREATE + mid-conversation change
RETRIEVELook up existing record
MODIFYUpdate existing record
DELETECancel/remove with confirmation
EDGEInvalid data, error handling
