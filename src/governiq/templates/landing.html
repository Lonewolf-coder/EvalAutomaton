How Evaluation Works

1
Candidate Submits Bot Export
The candidate exports their chatbot from the Kore.ai XO Platform as a JSON file and uploads it along with their Mock API URL.
2
Pipeline A — Structural Analysis (CBM Audit)
The engine parses the bot export and checks it against the assessment manifest: Does the bot have the required dialogs? Are all entity nodes present? Is the Agent Node configured? Are service nodes (POST, GET, PUT, DELETE) wired correctly? Are FAQs complete with alternate phrasings? This produces deterministic, reproducible results.
3
Pipeline B — Live Conversation Testing (Webhook Journey)
An LLM-powered conversation driver talks to the candidate's live bot through its webhook. It tests booking flows, retrieval, modification, cancellation, amendment handling, edge cases, and FAQs — exactly like a human QA tester would.
4
API State Verification
After each conversation, the State Inspector checks the candidate's Mock API to verify that data was actually persisted correctly. A bot can say all the right things but still fail to save data — this catches that.
5
Compliance Checks
Platform configuration is verified: Is DialogGPT enabled? Are required channels configured? Language settings correct? Critical compliance failures can override the overall score.
6
Detailed Report Generated
Both pipelines merge into a single scorecard. Every finding is explained in plain English. The candidate sees exactly what passed, what failed, and what to fix. The evaluator sees evidence cards and the CBM reference panel side by side.

Six Evaluation Patterns

Every task in every assessment maps to one of these patterns. The engine is domain-agnostic — it works the same for Travel, Medical, HR, or any future assessment type.

PatternWhat It TestsExample
CREATECollect entities and POST a new recordBook a flight, Schedule appointment
CREATE + AMENDSame as CREATE, but change one entity mid-conversation before confirmationBook flight, then change contact number before confirming
RETRIEVELook up an existing record by identifierGet appointment details by phone number
MODIFYRetrieve then update one or more fieldsChange travel date on existing booking
DELETECancel / remove a record and verify it's goneCancel a booking
EDGE CASEInject invalid values and verify graceful error handlingEnter fake phone number for retrieval

Scoring Breakdown

CBM Structural (40%)

Does the bot have the right structure? Correct dialogs, nodes, entities, service calls, and FAQs.

Webhook Functional (40%)

Does the bot actually work? Can it complete conversations, handle amendments, and persist data?

Compliance (10%)

Are platform settings correct? DialogGPT enabled, language configured, channels set up.

FAQs (10%)

Are all required FAQs implemented with correct answers and alternate phrasings?

Pass threshold: 70% overall score with no critical compliance failures.
