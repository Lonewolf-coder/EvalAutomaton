Quick Start

A manifest defines what to evaluate in a chatbot submission. The engine is domain-agnostic — the manifest provides all domain-specific configuration.

Required Fields
manifest_id, assessment_name, assessment_type, and at least one task.
Six Engine Patterns
CREATE, CREATE_WITH_AMENDMENT, RETRIEVE, MODIFY, DELETE, EDGE_CASE.
Save & Evaluate
Save your manifest and candidates can start submitting bots for evaluation.

Engine Patterns Reference

Task Patterns

CREATE
Tests entity collection, service call, confirmation flow. Bot should create a new record.
CREATE_WITH_AMENDMENT
CREATE + amendment: bot creates, user requests change, bot amends via agent node.
RETRIEVE
Tests lookup/search. Bot should retrieve and display existing records.

 

MODIFY
Tests update/change. Bot should modify an existing record.
DELETE
Tests deletion with confirmation. Bot should delete a record after user confirms.
EDGE_CASE
Tests error handling. Intentionally sends invalid data to test bot's error responses.
